# 에셋 제작 스펙

> 이 문서는 모든 2D 에셋의 제작 기준입니다.
> AI 이미지 생성 및 수작업 제작 시 이 스펙을 반드시 따릅니다.

---

## 1. 캐릭터 / 유닛 스프라이트

### 기본 규격

| 항목 | 값 |
|------|-----|
| 캔버스 크기 | 48 × 48 px |
| 파일 포맷 | PNG-32 (RGBA, 투명 배경) |
| 색상 모드 | RGBA |
| 해상도 | 72 dpi |
| 픽셀아트 대안 | 32 × 32 px |
| 고해상도 대안 | 64 × 64 px |

> **원칙**: 프로젝트 내 모든 캐릭터 스프라이트는 동일한 캔버스 크기를 사용한다.

### 애니메이션별 프레임 구성

| 애니메이션 | 프레임 수 | 루프 여부 | 설명 |
|-----------|----------|----------|------|
| `idle` | 4 ~ 6 | 루프 | 호흡, 미세 흔들림 |
| `walk` | 6 ~ 8 | 루프 | 이동 |
| `run` | 6 ~ 8 | 루프 | 빠른 이동 |
| `attack` | 4 ~ 8 | 비루프 | 공격 동작 |
| `hit` | 2 ~ 4 | 비루프 | 피격 반응 |
| `death` | 8 ~ 12 | 비루프 | 마지막 프레임 유지 |

### 스프라이트 시트 배열

```
행(row)   = 애니메이션 종류
열(col)   = 프레임 순서 (0부터 시작)

[idle_0  ][idle_1  ][idle_2  ][idle_3  ]
[walk_0  ][walk_1  ][walk_2  ][walk_3  ][walk_4  ][walk_5  ]
[attack_0][attack_1][attack_2][attack_3]
[hit_0   ][hit_1   ]
[death_0 ][death_1 ]...[death_11]
```

모든 프레임 크기는 동일해야 하며, 행마다 길이가 달라도 무방하다.

### 피벗 포인트 (Pivot Point)

| 유닛 종류 | x | y | 이유 |
|----------|---|---|------|
| 캐릭터 / 적 유닛 | 50% | 100% | 발바닥 중앙 기준으로 지면에 붙음 |
| 포탑 | 50% | 50% | 회전축이 중심이어야 정확히 회전 |
| 투사체 (총알) | 0% | 50% | 발사 기준점에서 앞으로 진행 |
| 폭발 이펙트 | 50% | 50% | 충돌 지점 중심으로 재생 |

### 콜라이더 (히트박스) 기준

| 유닛 종류 | 히트박스 크기 | 설명 |
|----------|-------------|------|
| 적 유닛 | 스프라이트의 60~70% | 작을수록 게임 체감 좋음 |
| 플레이어 포탑 | 스프라이트의 80% | |
| 총알 | 스프라이트와 거의 동일 (±2px) | |
| 방어 구조물 | 스프라이트의 90% | |

## 2. 타일셋

### 기본 규격

| 항목 | 값 |
|------|-----|
| 타일 1개 크기 | 32 × 32 px |
| 파일 포맷 | PNG-32 (투명 배경) |
| 시트 권장 크기 | 256 × 256 px (8×8 타일) |
| 타일 간격 | 0 px (여백 없음) |
| 픽셀아트 대안 | 16 × 16 px |

> **원칙**: 타일셋 내 모든 타일은 반드시 동일한 크기를 사용한다.

### 슈팅+디펜스 필수 타일 목록

| 타일 종류 | 설명 | 비고 |
|----------|------|------|
| 바닥 (기본) | 평지, 도로, 흙 등 | 적 이동 경로 |
| 바닥 (변형) | 균열, 얼룩 등 변형 3~4종 | 다양성 |
| 벽 / 장벽 | 상·하·좌·우·코너 | 8방향 세트 |
| 경로 표시 | 화살표, 점선 | 적 이동 방향 |
| 포탑 배치 영역 | 설치 가능 칸 | 하이라이트 포함 |
| 장식 (데코) | 풀, 돌, 나무 | 선택, 레이어 분리 |

### 타일셋 시트 배열

```
행 0: [바닥0    ][바닥1    ][바닥2    ][바닥3    ]
행 1: [벽_상   ][벽_하   ][벽_좌   ][벽_우   ]
행 2: [코너_TL ][코너_TR ][코너_BL ][코너_BR ]
행 3: [경로_→  ][경로_↓  ][경로_←  ][경로_↑  ]
행 4: [배치영역][하이라이트][빈칸    ][...]
```

### 메타데이터 JSON 구조

```json
{
  "tileWidth": 32,
  "tileHeight": 32,
  "columns": 8,
  "rows": 4,
  "tiles": [
    { "id": 0, "type": "ground", "walkable": true,  "name": "ground_dirt" },
    { "id": 4, "type": "wall",   "walkable": false, "name": "wall_top" },
    { "id": 8, "type": "path",   "walkable": true,  "name": "path_right" },
    { "id": 12,"type": "zone",   "walkable": false, "name": "placement_zone" }
  ]

```

## 3. 투사체 / 이펙트

### 투사체 규격

| 종류 | 크기 | 피벗 | 설명 |
|------|------|------|------|
| 기본 총알 | 8 × 8 px | x=0%, y=50% | |
| 큰 총알 | 16 × 16 px | x=0%, y=50% | |
| 미사일 | 16 × 32 px | x=0%, y=50% | 세로로 긴 형태 |
| 레이저 | 4 × 64 px | x=0%, y=50% | 가로 또는 세로 |

### 이펙트 스프라이트 시트 규격

| 이펙트 종류 | 프레임 수 | 캔버스 크기 | 루프 | 설명 |
|-----------|----------|-----------|------|------|
| 폭발 (소) | 8 프레임 | 64 × 64 px | 비루프 | 총알 히트 |
| 폭발 (대) | 12 프레임 | 128 × 128 px | 비루프 | 적 사망 |
| 히트 스파크 | 4 ~ 6 프레임 | 32 × 32 px | 비루프 | 충돌 순간 |
| 머즐 플래시 | 3 ~ 4 프레임 | 32 × 32 px | 비루프 | 발사 순간 |
| 연기 | 8 ~ 10 프레임 | 64 × 64 px | 비루프 | 소멸 후 연기 |

### 이펙트 재생 원칙

1. 모든 이펙트는 **비루프**로 설정한다.
2. 마지막 프레임 재생 완료 시 자동 소멸 (오브젝트 풀에 반납).
3. 오브젝트 풀링으로 관리 — 삭제/생성 반복 금지.
4. 충돌 위치 중심(x=50%, y=50%)에 스폰한다.

### 오브젝트 풀 권장 수량

| 풀 종류 | 권장 수량 |
|--------|---------|
| 총알 풀 | 30 ~ 50개 |
| 폭발 이펙트 풀 | 10 ~ 20개 |
| 히트 스파크 풀 | 10 ~ 15개 |

## 4. UI

### 해상도 기준

| 항목 | 값 |
|------|-----|
| 기준 해상도 | 390 × 844 px (iPhone 14) |
| 파일 포맷 | PNG-32 또는 SVG |
| 아이콘 크기 | 32 × 32 px |
| 버튼 최소 높이 | 44 px (터치 영역 확보) |

### 필수 UI 목록

| UI 요소 | 구성 | 위치 | 비고 |
|--------|------|------|------|
| HP 바 | 배경 + 게이지 2장 | 상단 HUD | 9-슬라이스 권장 |
| 쉴드 바 | 배경 + 게이지 2장 | HP 바 하단 | 옵션 |
| 웨이브 표시 | 아이콘 + 텍스트 | 상단 중앙 | Wave N / N |
| 코인 / 자원 | 아이콘 32×32 + 숫자 | 상단 우측 | |
| 포탑 선택 카드 | 썸네일 + 비용 + 이름 | 하단 | 스크롤 가능 |
| 게임오버 패널 | 배경 + 버튼 2~3개 | 중앙 오버레이 | |
| 승리 패널 | 배경 + 별 + 점수 | 중앙 오버레이 | |
| 일시정지 버튼 | 아이콘 44×44 | 우측 상단 | |

### 9-슬라이스 (9-Slice) 구조

크기가 변하는 UI 요소(버튼, HP바, 패널)는 9-슬라이스 방식으로 제작한다.

```
[TL][─ T ─][TR]   ← 모서리: 고정 크기
[│L][  M  ][R│]   ← 중앙: 늘어남
[BL][─ B ─][BR]   ← 모서리: 고정 크기
```

각 모서리 슬라이스 크기는 최소 8px 이상으로 제작한다.

## 5. 오디오

### 파일 포맷

| 종류 | 원본 | 배포 | 비고 |
|------|------|------|------|
| SFX (효과음) | WAV | OGG | 웹뷰 호환성 최우선 |
| BGM (배경음악) | WAV | MP3 or OGG | 루프 포인트 설정 필수 |

### 오디오 기술 규격

| 항목 | 값 |
|------|-----|
| 샘플레이트 | 44,100 Hz |
| 비트뎁스 | 16 bit |
| SFX 최대 길이 | 2.0 초 이하 |
| BGM 비트레이트 | 128 ~ 192 kbps |
| 피크 레벨 | -3 dBFS 이하 |
| 채널 (SFX) | 모노 (용량 절감) |
| 채널 (BGM) | 스테레오 |

### 필수 SFX 목록

|--------|------|
